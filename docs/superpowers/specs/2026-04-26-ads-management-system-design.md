# ADS Management System — Design Spec

**Date:** 2026-04-26  
**Status:** Approved  
**Phase:** 1 (dashboard + decision support) with Phase 2 hooks (Meta write operations)

---

## Overview

Sistema de gestão de anúncios que combina dados de conversão do Supabase, métricas de custo da Meta Ads API e enriquecimento de leads via HubSpot. Objetivo Phase 1: visibilidade completa de performance por campanha. Phase 2: execução de mudanças diretamente na Meta.

---

## Stack

| Camada | Tecnologia |
|--------|-----------|
| Frontend + API | Next.js 15 (App Router) |
| Banco de dados | Supabase (PostgreSQL) |
| Auth | Supabase Auth |
| Deploy + Cron | Vercel |
| Ads data | Meta Marketing API v20+ |
| CRM | HubSpot API |

---

## Fontes de Dados

### Existente (intocada)
```sql
public."[FH]Meta Ads" (
  id, created_at, ads_id, phone_client, click_id,
  source_url, phone_company, data_sent,
  campaign_id, campaign_name, adset_id, adset_name, ad_name
)
```
Source of truth de conversões/leads com atribuição Meta.

### Novas tabelas

```sql
-- Snapshot diário de métricas Meta Ads
meta_insights (
  id            bigint generated by default as identity primary key,
  date          date not null,
  campaign_id   text not null,
  campaign_name text,
  adset_id      text not null,
  adset_name    text,
  ad_id         text not null,
  ad_name       text,
  spend         numeric(10,2),
  impressions   integer,
  clicks        integer,
  reach         integer,
  synced_at     timestamp with time zone default now(),
  unique (date, campaign_id, adset_id, ad_id)
)

-- Contatos HubSpot enriquecidos, match por telefone
hubspot_contacts (
  id              bigint generated by default as identity primary key,
  phone           text not null unique,
  hs_contact_id   text,
  lifecycle_stage text,
  deal_value      numeric(10,2),  -- valor do deal aberto mais recente
  deal_stage      text,
  updated_at      timestamp with time zone default now()
)

-- Log de sincronizações
sync_logs (
  id              bigint generated by default as identity primary key,
  type            text not null, -- 'meta' | 'hubspot'
  status          text not null, -- 'success' | 'error'
  message         text,
  records_synced  integer,
  created_at      timestamp with time zone default now()
)
```

## Fluxo de Dados

```
Meta Ads API ──(Vercel Cron, diário 06:00 UTC)──→ meta_insights
                                                        │
"[FH]Meta Ads" ────────────────────────────────────────┼──→ Dashboard
                                                        │    (join campaign_id)
HubSpot API ──(on-demand, match por phone_client)──→ hubspot_contacts
                                                        │
                                                        ↓
                                              CPL, ROAS, funil
```

### Métricas calculadas

| Métrica | Fórmula |
|---------|---------|
| CPL | `spend ÷ leads` |
| CTR | `clicks ÷ impressions` |
| ROAS | `deal_value ÷ spend` |
| Leads | `count([FH]Meta Ads) por campaign_id` |

## Páginas

### `/dashboard`
- KPIs globais: total spend, total leads, CPL médio, ROAS
- Filtro de período: 7d / 30d / custom
- Gráfico: leads + spend ao longo do tempo

### `/campaigns`
- Tabela: campanha → spend / leads / CPL / CTR / impressões
- Drill-down: campanha → adsets → ads
- Badges de alerta: CPL acima da média, CTR baixo

### `/leads`
- Lista de leads de `[FH]Meta Ads` enriquecida com HubSpot
- Colunas: telefone, campanha, adset, ad, status HubSpot, valor deal
- Filtros por campanha e período

### `/settings`
- Meta Ads: token + Ad Account ID
- HubSpot: API key
- Trigger manual de sync
- Status do último sync (via sync_logs)

## API Routes

```
POST /api/meta/sync           ← Vercel Cron (diário) — protegido por CRON_SECRET
POST /api/hubspot/enrich      ← enriquece lote de leads por telefone
GET  /api/insights/campaigns  ← agrega dados pro dashboard
```

### Sync Meta Ads (detalhe)
1. Primeira execução: janela de 90 dias. Execuções subsequentes: 30 dias.
2. Chama Meta Marketing API → insights da janela definida
3. Upsert em `meta_insights` por `(date, campaign_id, adset_id, ad_id)`
3. Registra resultado em `sync_logs`
4. Token expirado → alerta no dashboard + notificação

## Error Handling

| Cenário | Tratamento |
|---------|-----------|
| Token Meta expirado | Alerta visual no dashboard + badge em `/settings` |
| HubSpot rate limit | Retry com exponential backoff (max 3x) |
| Sync falhou | Último sync bem-sucedido visível em `/settings` via `sync_logs` |
| Dados Meta ausentes | Dashboard mostra lead count mas spend como "sem dados" |
